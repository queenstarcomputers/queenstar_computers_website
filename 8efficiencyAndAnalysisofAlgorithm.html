Efficiency (Performance) and Analysis of an Algorithm

An algorithm is a sequence of computational steps written in order to carry out some particular tasks. Each step of an algorithm is written in English. Data structures and algorithms are the building blocks of computer programming.

Efficiency (Performance) of an Algorithm

Efficiency is one of the major measures of performance or efficiency of an algorithm. Efficiency is how fast an algorithm can produce correct results for the given problem. It depends on:

a) Time complexity (to save time): It is a way of representing the amount of computer time (compile and run time) needed by a program to run to the completion (execute). The compile time of an algorithm does not depend on the instance characteristics of the algorithm. The run time of an algorithm is estimated by determining the number of various operations, such as addition, subtraction, multiplication, division, load and store executed by it. A program that runs faster is a better program.

b) Space complexity (to save space): It is the amount of memory space required by an algorithm, during a course of its execution until final output is generated. Space complexity is required in situations when limited memory is available and for the multi user system. It is composed of two different spaces; Auxiliary space and Input space. A program that saves space over a computing program is considerable and desirable. A function of space needed by an algorithm can be computed by either an empirical or theoretical approach. The empirical testing approach call for implementing the complete algorithm and executing them on a computer for various instances are noted and compared. The algorithm whose implementation yields the least time is considered as the best among the candidate algorithm.

Analysis of an Algorithm

Analysis of an Algorithm determines the amount of resources (like time and space) it requires for its execution. Generally, the algorithms are formulated to work with the inputs or arbitrary length. Algorithm analysis provides theoretical estimates required by an algorithm to solve a problem. In computer science, the analysis of algorithms is the process of finding the computational complexity of algorithms i.e., the amount of time, storage, or other resources needed to execute them. We analyze algorithms using a mathematical notation for functions that disregards constant factors i.e., we characterize the running times of algorithms by using functions that map the size of the input, n, to values that correspond to the main factor that determines the growth rate in terms of n. This approach allows us to focus on the big-picture aspects of an algorithm's running time.

Why Analyze an Algorithm?

  • To discover its characteristics in order to evaluate its suitability for various applications or compare it with other algorithms for the same application.
  • It helps us understand it better, and can suggest informed improvements.
  • Algorithms tend to become shorter, simpler, and more elegant during the analysis process.
